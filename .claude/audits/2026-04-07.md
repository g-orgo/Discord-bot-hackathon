## Issues Found

### Major

**1. `raptor-chatbot-server` documentation described wrong architecture** — SEVERITY: Major
- Both `CLAUDE.md` and `.github/copilot-instructions.md` described the server as a "single-file Express application" with an "in-memory USERS array" — but the server has been fully migrated to MongoDB + Mongoose with a multi-file structure (`src/routes/`, `src/models/`, `src/middleware/`).
- This misinformation would cause any AI assistant to generate incorrect code (adding in-memory state, single-file changes, missing routes, wrong return types).
- **Fixed:** Both files rewritten with correct architecture, full endpoint table, accurate conventions.

**2. `raptor-chatbot-server` endpoint table was incomplete** — SEVERITY: Major (linked to above)
- Endpoints table showed only 4 endpoints; actual server has 10 endpoints including history CRUD, SSE stream, Discord bot route, and profile update.
- **Fixed:** Full endpoint table added.

---

### Minor

**3. Dead code: `logChannelMessages()` in `raptor-chatbot/api/discord.js`** — SEVERITY: Minor
- `logChannelMessages` was exported but not imported by any current caller (`commandHandler.js` was never updated to use it; `app.md` context incorrectly said it was called for a `logchannel` command that was never registered).
- **Fixed:** Function removed from `discord.js`.

**4. Stale context `app.md` imports** — SEVERITY: Minor
- Listed `InteractionResponseFlags`, `MessageComponentTypes`, and `logChannelMessages` as imports in `app.js` — none of these are imported in the actual file.
- **Fixed:** Imports section updated to match actual `app.js`.

**5. Stale context `commands.md` — listed unregistered commands** — SEVERITY: Minor
- Listed `/logchannel` as a registered command. Not in `commands.js`. Also listed `/test` which was removed.
- **Fixed:** Registered commands updated to match actual `commands.js` (`/ask`, `/translatechannel`, `/clearchannel`).

**6. `raptor-chatbot-llm` copilot-instructions missing `/chat/stream` and new env vars** — SEVERITY: Minor
- Endpoint table missing `POST /chat/stream` (added in a recent session). Env vars `OLLAMA_TIMEOUT` and `OLLAMA_PULL_TIMEOUT` not documented.
- **Fixed:** Endpoint added; env vars documented.

**7. `raptor-chatbot-llm` context `main.md` described as single file** — SEVERITY: Minor
- Architecture section said "Single file: `main.py`" but the app is now split into `config.py`, `ollama.py`, `schemas.py`, `system_prompt.py`, and `routes/`.
- **Fixed:** Architecture section rewritten to reflect actual file structure.

## No Issues Found In

- **raptor-chatbot-web** — All URLs relative, sessionStorage used, props flowing correctly, no inline styles, route guards in place.
- **raptor-chatbot-llm** code — Pydantic models used, async routes, CORS via env, input length validation via Field constraints.
- **raptor-chatbot-server** code — bcrypt hashing, JWT from env, rate limiting on auth routes, input validation at route level, ESM throughout.
- **raptor-chatbot** command handling — Components V2 compliance, deferred responses with fire-and-forget, error handling inside try/catch.
